# 产品需求文档 (PRD)

## 项目概述

**项目名称**: Global Macro Finance API (全球宏观经济债券利率数据服务)

**版本**: v1.0.0

**最后更新**: 2026-03-03

---

## 1. 产品定位

### 1.1 产品描述
全球宏观经济债券利率数据服务是一个后端微服务，负责从 FRED API 获取美国、德国、日本等主要经济体的国债收益率数据，为前端提供实时数据查询接口，支持 n8n 自动化工作流定时更新数据。

### 1.2 目标用户
- 前端应用：展示宏观经济债券利率曲线图
- n8n 工作流：定时触发数据更新
- 监控系统：健康检查和数据质量监控

---

## 2. 核心功能

### 2.1 数据获取功能

#### 2.1.1 美国国债数据
| 功能 | 描述 |
|------|------|
| **数据类型** | 3个月、2年、10年期国债收益率 |
| **数据频率** | 每日更新 |
| **历史范围** | 从 2000-01-01 开始 |
| **数据来源** | FRED API (DGS3MO, DGS2, DGS10) |

#### 2.1.2 OECD 债券数据
| 功能 | 描述 |
|------|------|
| **数据类型** | 德国、日本 10 年期国债收益率 |
| **数据频率** | 月度更新 |
| **历史范围** | 最近 365 天 |
| **数据来源** | FRED API (IRLTLT01DEM156N, IRLTLT01JPM156N) |

### 2.2 数据更新功能

#### 2.2.1 历史数据获取
- **接口**: `POST /api/fetch/us-treasuries/history`
- **用途**: 首次部署或重建数据时获取完整历史数据
- **行为**: 从 2000-01-01 开始获取全部美债历史数据
- **响应时间**: 约 30-40 秒

#### 2.2.2 增量更新
- **接口**: `POST /api/update/us-treasuries`
- **用途**: n8n 定时调用，更新最近 7 天数据
- **行为**: 获取最近 7 天的美债数据并合并到本地存储
- **响应时间**: 约 3-5 秒

#### 2.2.3 完整更新
- **接口**: `POST /api/update`
- **用途**: 更新美债和 OECD 债券数据
- **行为**: 美债（最近 7 天）+ OECD（最近 365 天）
- **响应时间**: 约 10-15 秒

### 2.3 数据查询功能

#### 2.3.1 时间范围查询
- **接口**: `GET /api/data`
- **参数**:
  - `start_date`: 起始日期 (YYYY-MM-DD)
  - `end_date`: 结束日期 (YYYY-MM-DD)
- **默认**: 返回最近 90 天数据
- **用途**: 前端展示曲线图

### 2.4 健康检查功能

#### 2.4.1 服务状态
- **接口**: `GET /api/health`
- **返回**: 服务状态、版本号、最后更新时间
- **用途**: 监控系统探测

## 4. 数据质量要求

### 4.1 数据准确性
- 所有数据必须来自官方 FRED API
- 数据采集失败需要记录日志
- 异常值需要标记为 null

### 4.2 数据完整性
- 美债数据需要覆盖 2000-01-01 至今
- OECD 数据至少覆盖最近 365 天
- 缺失数据使用 null 填充

### 4.3 数据时效性
- 美债数据每日更新
- OECD 数据月度更新
- 更新失败需要重试机制

### 5.1 性能要求
| 接口 | 响应时间 | 并发 |
|------|----------|------|

| 增量更新 | < 5 秒 | 1 |
| 完整更新 | < 15 秒 | 1 |

- 支持新增数据源
- 支持接口版本管理

## 6. 技术约束

### 6.1 数据存储
- 使用 CSV 文件存储历史数据
- 支持增量追加
- 数据文件位置: `./data/`

### 6.2 外部依赖
- FRED API（需要 API Key）
- 网络连接稳定
- API 调用频率限制

### 6.3 部署环境
- 端口: 8094
- Python 3.14+
- 需要 .env 配置文件

## 7. 用户场景

### 7.1 首次部署场景
1. 启动服务
2. 调用 `/api/fetch/us-treasuries/history` 获取历史数据
3. 等待 30-40 秒完成数据获取
4. 前端可以展示完整历史曲线

### 7.2 日常更新场景
1. n8n 每日凌晨 2 点触发
2. 调用 `/api/update/us-treasuries` 更新数据
3. 3-5 秒内完成更新
4. 前端自动展示最新数据

### 7.3 前端查询场景
1. 用户打开前端页面
2. 前端调用 `/api/data?start_date=2000-01-01`
3. 获取完整历史数据
4. 渲染债券收益率曲线图
